# Glossary of Terms

## Core Concepts

### A
- **A2A Protocol**: Agent-to-Agent Protocol - Google's open standard for inter-agent communication
- **A2A-R**: A2A Robotics extensions - Real-time, safety-critical messaging for robotics
- **Actualizer.ai**: Multi-horizon planning tool working backward from 1-100 years simultaneously
- **Agent Card**: Structured metadata document describing an agent's capabilities

### C
- **Capitaine**: Git-native agent framework where the repository IS the agent
- **Cocapn**: The software vessel - a meta-agent that orchestrates other agents on a physical vessel
- **Competence Cliff**: The point where accumulated capabilities exceed user's ability to understand them
- **Context Repository**: Git-based memory system for agents (from Letta)

### E
- **Embodied Foundation Model**: AI model trained for physical robot interaction
- **Experience Primitive**: Atomic unit of learning - context, action, outcome, learning signal

### F
- **Field-Captain**: Jetson field agent with voice, local LLM, and git-agent orchestration
- **Fleet-Native**: Designed from the ground up for multi-agent coordination
- **FP4 Quantization**: 4-bit floating point for efficient edge AI inference

### G
- **Git-Native Agent**: An agent where the repository IS the agent body and git history IS memory
- **Glass Box Principle**: All agent reasoning must be human-inspectable

### I
- **Innovation Portfolio**: A collection of verified, attributed, valuable contributions that compound over time
- **Innovation Density Index (IDI)**: Metric measuring innovations per vessel per month

### J
- **Jetson Thor**: NVIDIA's edge AI platform for physical AI and humanoid robotics (2070 FP4 TFLOPS)

### K
- **Knowledge Graph**: Structured representation of capabilities and their relationships

### L
- **LoRA**: Low-Rank Adaptation - efficient fine-tuning method for large models

### M
- **MIG**: Multi-Instance GPU - Jetson Thor feature for partitioning GPU among agents

### N
- **Nexus-Runtime**: Orchestration layer for coordinating robotic cells and fleets

### O
- **Offline-First**: Architecture that operates fully without network connectivity

### P
- **Portfolio Appreciation Rate (PAR)**: Growth rate of portfolio value over time
- **Portfolio Economy**: Economic system where innovation portfolios are the primary value unit

### R
- **Reverse Actualization**: Planning methodology that starts from desired future and works backward to find mandatory prerequisites
- **ROM-Emulation Architecture**: Hybrid of stable base layer + flexible application layer

### S
- **Self-Improving Agent**: Agent that iteratively refines capabilities through experience
- **Software Vessel**: The cocapn agent system running on a physical vessel

### V
- **Vessel**: Any physical system - boat, drone, robot, home assistant, medical device
- **Vessel Bridge**: Interface layer between software vessel and physical vessel

## Metrics

- **ECM (Embodiment Completeness Metric)**: Degree to which agent has complete physical interaction
- **FCC (Fleet Coordination Coefficient)**: Effectiveness of multi-agent coordination
- **HER (Human Engagement Resilience)**: Ability to maintain user participation across experience levels
- **RAF (Reverse Actualization Fidelity)**: Alignment between planned path and actual outcomes
- **SPS (Sovereignty Preservation Score)**: Degree to which humans retain meaningful control
- **TPFI (Technical Paradigm Feasibility Index)**: Gap between aspiration and implementable capability

## Failure Severities

- **SEV-0**: Catastrophic - Safety incident, data loss, fleet-wide failure (RTO: Immediate)
- **SEV-1**: Critical - Vessel inoperable, capability unavailable (RTO: <1 hour)
- **SEV-2**: Major - Degraded operation, workaround exists (RTO: <4 hours)
- **SEV-3**: Minor - Non-critical feature affected (RTO: <24 hours)
- **SEV-4**: Low - Cosmetic, documentation (RTO: <1 week)

## QoS Levels (A2A-R)

- **Safety-Critical**: <10ms latency, 99.999% delivery guarantee
- **Real-Time**: <100ms latency, 99.9% delivery guarantee
- **Best-Effort**: No latency or delivery guarantees

## Human Roles

- **Installer**: Deploys and configures vessels
- **Educator**: Trains users and develops learning materials
- **Technician**: Diagnoses issues and performs maintenance
- **Portfolio Curator**: Reviews and curates innovations (emergent role)
- **Ethics Auditor**: Ensures alignment with human values (emergent role)
- **Agent Trainer**: Specializes in capability refinement (emergent role)

## Vessel Types

- **Marine Vessel**: Boat, ship, watercraft
- **Aerial Vessel**: Drone, UAV, aircraft
- **Industrial Vessel**: Warehouse robot, manufacturing system
- **Home Vessel**: Personal assistant, chef robot, cleaner
- **Medical Vessel**: Surgical robot, diagnostic system
